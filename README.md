# showcase-subway-accessibility

Thin portfolio wrapper around the
[`subway-access` v0.5.0](https://github.com/random-walks/subway-access)
CASESTUDY. The canonical analysis lives upstream; this showcase adds
blaise-website-specific framing, a browsable gallery of the 15
committed figures, and a cross-walk mapping the upstream sections to
`factor-factory` engine families.

## Five-bullet summary

- **Subject**: ADA accessibility of the NYC subway system, April 2026
  vintage — 493 stations, 157 ADA-accessible (31.8%), 2,317 tracts,
  8.5M resident study population.
- **Headline finding**: 4,717,140 New Yorkers (55.4%) live in tracts
  with no ADA-accessible station within an 800 m walk; Queens
  (1.75M gap population) and Staten Island (9% coverage) are the
  most underserved.
- **Methods mirrored from upstream**: OLS equity regression
  (*R*² = .202, *F*(3, 2313) = 108.83, *p* < .001), Moran's *I* for
  spatial clustering (*I* = .23 for gap score, *z* = 40.87), 7-period
  tract × year panel with sourced ADA upgrade dates for 101 / 157
  stations.
- **Reliability twist**: 49 of 157 ADA stations operated below 95%
  elevator uptime over the 12-month observation window; Manhattan's
  coverage drops from 77% nominal to 71% effective when uptime-weighted.
- **Status**: not recomputed here. Figures + upstream reference are
  the artifacts. Regeneration requires cloning the upstream repo —
  see `manuscripts/UPSTREAM_REFERENCE.md`.

## Run it

From the repo root:

```bash
pnpm showcase:run showcase-subway-accessibility/notebooks/01_context_and_pointer.py
pnpm showcase:run showcase-subway-accessibility/notebooks/02_figures_gallery.py
pnpm showcase:run showcase-subway-accessibility/notebooks/03_cross_walk.py

pnpm showcase:render showcase-subway-accessibility
pnpm showcase:lint   showcase-subway-accessibility
pnpm showcase:view   showcase-subway-accessibility
```

`jellycell.toml` pins the project name so the `--project` flag in the
wrapper scripts resolves correctly (per jellycell #15 workaround).
